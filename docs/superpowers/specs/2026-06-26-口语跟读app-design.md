# 口语跟读 App 设计文档

- 日期：2026-06-26
- 平台：iPhone（iOS 原生）
- 状态：设计已确认，待写实现计划

## 1. 背景与定位

中文是表意文字，看字即可理解；英语是表达性语言，日常沟通靠固定的口语格式。
掌握常用口语句型即可应对约 80% 的日常沟通。本 app 帮助用户通过
"录原声 → 反复听 → 跟读 → 自我对比" 来练习并内化这些口语格式。

一句话定位：iPhone 上的"美剧台词跟读"工具——录下原声、配上台词、反复听、
自己跟读，靠耳朵自我对比。

## 2. 核心范围（MVP）

一个**片段（Clip）= 最小练习单元**，包含：

- 🎙️ 原声录音：手机麦克风外放录下的对话
- 📝 台词文字：手动输入/粘贴的英文台词（核心，用来记句型）
- 🗣️ 多次跟读（Attempt）：照着读、录下来的多个版本，保留历史

典型流程：

1. 电脑/电视放着美剧 →（app 内）录下这句对话 → 停止
2. 配上台词文字，起名，（可选）归入合集
3. 进入练习页：循环听原声 → 录我的跟读 → 原声/我的版本连续对比播放
4. 不满意就再录一次（新增一条历史），满意就练下一句

### 明确不做（保持聚焦）

- ❌ 自动发音评分 / 语音转文字比对（用户决定先不做）
- ❌ 直接抓取 Netflix 等 app 的系统声音（iOS 限制，改用外放麦克风录制）
- ❌ 账号、云同步、社交（先纯本地单机）
- ❌ 自带素材库（暂不预置内容，避免版权问题）

## 3. 数据结构（本地存储）

```
Collection（合集，可选分组）
  - name
  - clips: [Clip]

Clip（片段，核心单元）
  - title          名字，如 "How you doin'"
  - scriptText     台词文字（手动输入/粘贴）
  - originalAudio  原声录音文件路径
  - attempts: [Attempt]   一对多，保留历史
  - createdAt
  - lastPracticedAt

Attempt（一次跟读）
  - audio          录音文件路径
  - createdAt      录制时间
  -（预留：以后可挂自评分/打星）
```

- 元数据用 **SwiftData** 存储；录音文件存 app 沙盒 Documents 目录，
  数据库只存文件路径。
- 「我的跟读」保留多次历史，可回放任意一次、与原声对比、删除不满意的，
  以便对比不同时间的进步。

## 4. 界面（4 屏）

1. **片段库（首页）**
   按合集分组的 Clip 列表，每行显示名字 + 台词首句。右上角「＋ 录新片段」。

2. **录制页**
   大录音按钮（开始/停止），录完可试听、可重录。下方填台词、起名、
   选合集 → 保存。

3. **练习页（最核心）**
   ```
   ┌─────────────────────────┐
   │  台词文字（大字显示）        │
   │  "How you doin'?"        │
   ├─────────────────────────┤
   │  ▶ 原声      🔁 循环       │
   │  ●  录我的跟读             │
   │  ▶ 我的版本（最近一次）     │
   ├─────────────────────────┤
   │  ⇄  原声→我的 连续对比播放  │
   ├─────────────────────────┤
   │  历史跟读：3 次 ▸          │
   └─────────────────────────┘
   ```
   一屏内完成「听原声 → 录自己 → 对比」闭环；底部历史跟读列表可回放/删除。

4. **片段详情/编辑**
   改台词、改名、删除（可先合并进练习页）。

## 5. 技术选型

- **SwiftUI** — 界面
- **SwiftData** — Clip / Attempt / Collection 元数据持久化
- **AVFoundation**（`AVAudioRecorder` / `AVAudioPlayer`）— 录音、循环播放、
  连续对比播放
- 录音文件存 app 沙盒 Documents 目录，数据库只存路径
- 最低支持 **iOS 17**（SwiftData 要求）
- 无后端、无联网

## 6. 测试策略

- 数据层（Clip/Attempt/Collection 的增删改查、文件路径读写）写单元测试。
- 录音/播放依赖硬件，封装为 `AudioService` 协议：逻辑部分可单元测试，
  录放在真机手动验证。

## 7. 第一个里程碑（最小可跑闭环）

建 1 个片段 → 录原声 → 填台词 → 循环听 → 录我的跟读 → 对比播放。
先不做合集分组与独立编辑页，跑通主流程后再补。
